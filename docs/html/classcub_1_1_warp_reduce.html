Detailed description

template<typename T, int WARPS = 1, int LOGICAL_WARP_THREADS = PtxArchProps::WARP_THREADS>
class cub::WarpReduce< T, WARPS, LOGICAL_WARP_THREADS >

WarpReduce provides variants of parallel reduction across CUDA warps.

.
Overview
A reduction (or fold) uses a binary combining operator to compute a single aggregate from a list of input elements.
For convenience, WarpReduce exposes a spectrum of entrypoints that differ by:
  • Operator (generic reduction vs. summation for numeric types)
  • Input (full warps vs. partially-full warps having some undefined elements)
Template Parameters
TThe reduction input/output element type
WARPS[optional] The number of "logical" warps performing concurrent warp reductions. Default is 1.
LOGICAL_WARP_THREADS[optional] The number of threads per "logical" warp (may be less than the number of hardware warp threads). Default is the warp size of the targeted CUDA compute-capability (e.g., 32 threads for SM20).
Usage Considerations
  • Supports non-commutative reduction operators
  • Supports "logical" warps smaller than the physical warp size (e.g., a logical warp of 8 threads)
  • The number of entrant threads must be an even multiple of /p LOGICAL_WARP_THREADS (default is the warp size of the targeted CUDA compute-capability)
  • Warp reductions are concurrent if more than one warp is participating
  • The warp-wide scalar reduction output is only considered valid in warp-lane0
  • After any operation, a subsequent __syncthreads() barrier is required if the supplied WarpReduce::SmemStorage is to be reused or repurposed by the threadblock
Performance Considerations
  • Uses special instructions when applicable (e.g., warp SHFL)
  • Uses synchronization-free communication between warp lanes when applicable
  • Zero bank conflicts for most types.
  • Computation is slightly more efficient (i.e., having lower instruction overhead) for:
    • Reduction parameterizations where T is a built-in C++ primitive or CUDA vector type (e.g., short, int2, double, float2, etc.)
    • Reduction parameterizations where LOGICAL_WARP_THREADS is a multiple of the architecture's warp size
Algorithm
These parallel reduction variants implement a warp-synchronous Kogge-Stone algorithm having O(logn) steps and O(nlogn) work complexity, where n = LOGICAL_WARP_THREADS (which defaults to the warp size associated with the CUDA Compute Capability targeted by the compiler).

Data flow within a 16-thread Kogge-Stone reduction construction. Junctions represent binary operators.

Examples
Example 1. Perform a simple sum reduction for one warp
#include <cub/cub.cuh>
__global__ void SomeKernel(...)
{
// Parameterize WarpReduce for 1 warp on type int
typedef cub::WarpReduce<int, 1> WarpReduce;
// Opaque shared memory for WarpReduce
__shared__ typename WarpReduce::SmemStorage smem_storage;
// Perform a reduction of thread ranks in first warp
if (threadIdx.x < 32)
{
int input = threadIdx.x;
int output = WarpReduce::Sum(smem_storage, input);
if (threadIdx.x == 0)
printf("tid(%d) output(%d)\n\n", threadIdx.x, output);
}
...
Printed output:
tid(0) output(496)
