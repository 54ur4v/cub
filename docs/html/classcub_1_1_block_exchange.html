template< typename T, int BLOCK_THREADS, int ITEMS_PER_THREAD, bool WARP_TIME_SLICING = false>
class cub::BlockExchange< T, BLOCK_THREADS, ITEMS_PER_THREAD, WARP_TIME_SLICING >

BlockExchange provides operations for reorganizing the partitioning of ordered data across a CUDA threadblock.

.
Overview
BlockExchange allows threadblocks to reorganize data items between threads. More specifically, BlockExchange supports the following types of data exchanges:
  • Transposing between blocked and striped arrangements
  • Transposing between blocked and warp-striped arrangements
  • Scattering to a blocked arrangement
  • Scattering to a striped arrangement
Template Parameters
TThe data type to be exchanged.
BLOCK_THREADSThe threadblock size in threads.
ITEMS_PER_THREADThe number of items partitioned onto each thread.
WARP_TIME_SLICING[optional] When true, only use enough shared memory for a single warp's worth of tile data, time-slicing the block-wide exchange over multiple synchronized rounds (default = false)
Algorithm
Threads scatter items by item-order into shared memory, allowing one item of padding for every memory bank's worth of items. After a barrier, items are gathered in the desired arrangement.
A threadblock of 16 threads reading a blocked arrangement of 64 items in a parallel "raking" fashion.
Usage Considerations
  • After any operation, a subsequent __syncthreads() barrier is required if the collective's BlockExchange::TempStorage is to be reused or repurposed
Performance Considerations
  • Proper device-specific padding ensures zero bank conflicts for most types.

typedef _TempStorage TempStorage
 The operations exposed by BlockExchange require a temporary memory allocation of this type for thread communication. This opaque storage can be allocated directly using the __shared__ keyword. Alternatively, it can be aliased to externally allocated memory (shared or global) or union'd with other storage allocation types to facilitate memory reuse.
