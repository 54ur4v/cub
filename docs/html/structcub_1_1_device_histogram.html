cub::DeviceHistogram Struct Reference
Device-wide

Detailed description

DeviceHistogram provides device-wide parallel operations for constructing histogram(s) from a sequence of samples data residing within global memory.

.
Overview
A histogram counts the number of observations that fall into each of the disjoint categories (known as bins).
Usage Considerations
  • Dynamic parallelism. DeviceHistogram methods can be called within kernel code on devices in which CUDA dynamic parallelism is supported. When calling these methods from kernel code, be sure to define the CUB_CDP macro in your compiler's macro definitions.

inlinestatic

Computes an intensity histogram from a sequence of data samples using equal-width bins.

  • The number of histogram bins is num_levels - 1
  • The value ranges for all bins have the same width: (upper_level - lower_level) / (num_levels - 1)
  • When d_temp_storage is NULL, no work is done and the required allocation size is returned in temp_storage_bytes.
  • When calling this method from kernel code, be sure to define the CUB_CDP macro in your compiler's macro definitions.

inlinestatic

Computes intensity histograms from a sequence of multi-channel data samples using equal-width bins.

  • The input is a set of independent data channels that have been interleaved into a single sequence of data samples (e.g., an array of pixels where each pixel is comprised of four RGBA sample values). The total number of samples across all channels (num_samples) must be a whole multiple of NUM_CHANNELS.
  • Of the NUM_CHANNELS specified, the function will only compute histograms for the first NUM_ACTIVE_CHANNELS (e.g., RGB histograms from RGBA pixel samples).
  • The number of histogram bins for channeli is num_levels[i] - 1.
  • For channeli, the value ranges for all histogram bins have the same width: (upper_level[i] - lower_level[i]) / ( num_levels[i] - 1)
  • When d_temp_storage is NULL, no work is done and the required allocation size is returned in temp_storage_bytes.
  • When calling this method from kernel code, be sure to define the CUB_CDP macro in your compiler's macro definitions.
Snippet

unsigned int upper_level[3]; // e.g., {256, 256, 256};

// [0, 0, 2, 0, 0, 0, 1, 2, 0, 0, 0, ..., 0] ]
